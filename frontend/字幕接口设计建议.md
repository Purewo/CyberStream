# 字幕样式参数持久化后端接口建议

## 1. 背景与需求
目前前端播放器的字幕样式参数（例如中英文字号、颜色、行间距、垂直偏移）保存在用户的浏览器 `localStorage` 中（全局生效）。
然而不同的影片（比如有硬字幕的电影需要更大的 offset 去避开，或者不同的默认分辨率、用户偏好等）需要有自己独立的字幕设定。
因此，建议后端提供一对专门保存和读取**影片级别**（或者**用户影片级别**，如果支持多用户的话）的字幕样式参数的接口。

## 2. 数据结构设计 (SubtitleSettings)
前端目前需要的完整的字幕样式配置表如下：

```typescript
interface ResourceSubtitleSettings {
  zhSize?: number;     // 中文字幕字号大小，默认 24
  zhColor?: string;    // 中文字幕颜色，默认 "#ffffff"
  enSize?: number;     // 英文字幕字号大小，默认 18
  enColor?: string;    // 英文字母颜色，默认 "#ffffff"
  gap?: number;        // 中英文字幕之间的行间距，默认 12
  offset?: number;     // 字幕整体在底部的垂直偏移量（调整避开内嵌硬字幕），默认 -4
}
```

## 3. 推荐的接口设计

### 3.1 获取当前资源的字幕设置
**接口：** `GET /api/v1/resources/{id}/subtitle-settings`
**说明：** 前端在加载影片（或播放器初始化）时调用，用于获取该影片缓存的字幕样式配置。
**响应结果 (200 OK)：**
```json
{
  "code": 200,
  "data": {
    "zhSize": 24,
    "zhColor": "#ffffff",
    "enSize": 18,
    "enColor": "#ffffff",
    "gap": 12,
    "offset": -4
  }
}
```
*注：如果该资源以前没有特殊配置，后端也可以返回 404，或者返回全空的/带有默认值的对象，前端会处理缺省状态。*

### 3.2 保存/更新字幕设置
**接口：** `PUT /api/v1/resources/{id}/subtitle-settings`  (或者用 `PATCH`)
**说明：** 当用户在前端的“字幕设置”面板中拖动滑块改变了任意样式参数并确认时调用，将最新数据同步至后端。
**请求体 (Request Body)：**
```json
{
  "zhSize": 26,
  "offset": 20
}
```
*注：建议后端支持局部更新（Partial Update），即前端只传发生改变的字段，后端将其合并（Merge）进原本的设置数据中并落库。*

## 4. 存储方案建议
1. **依附于 Resource/Playback 指针：** 如果资源数据字典允许，可以直接在 `playback.subtitles` 下增加一个 `settings` 节点：
   ```json
   "playback": {
      "subtitles": {
         "default_subtitle_id": "...",
         "items": [...],
         "settings": { "zhSize": 24, "offset": -4 } 
      }
   }
   ```
   **优势**：可以不用增加新的独立接口，前端只需要在现有的拉取接口里获取即可；不过依然需要一个专门更新这段 setting 结构体的接口进行保存，避免改写播放列表的其他数据。
2. **依附于 User + Resource：** 若为多用户系统，强烈建议此配置做成 `User -> Resource` 的属性关联实体（类似于该用户在这部影片上的播放进度），避免其他用户观影时受到该用户字幕偏移的影响。
